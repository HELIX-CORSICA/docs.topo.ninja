## Résumé

Classe les points importés en deux catégories (projet et implantés) via glisser-déposer ou détection automatique de marqueurs dans les noms de points. Interface à trois colonnes : projet, pool central, implantés.

## Principe

Après l'import CSV, tous les points arrivent dans le pool central (colonne du milieu). L'objectif est de les répartir dans les deux colonnes latérales : points projet (coordonnées théoriques) à gauche, points implantés (coordonnées mesurées) à droite. La détection automatique identifie les marqueurs d'implantation dans les noms de points. Les points restants sont déplacés manuellement par glisser-déposer.

## Étapes d'utilisation

**Étape 1 -- Importer les points**
Importer les points via CSV ou saisie manuelle. Tous les points apparaissent dans le pool central.

**Étape 2 -- Lancer la détection automatique**
Activer la détection automatique. L'algorithme analyse les noms de points et identifie ceux contenant des marqueurs d'implantation (stkd, stked, imp). Ces points sont déplacés vers la colonne "Points Implantés".

**Étape 3 -- Compléter manuellement**
Pour les points non détectés automatiquement, les sélectionner et les glisser vers la colonne correspondante.

**Étape 4 -- Valider**
Vérifier que chaque colonne contient les bons points, puis valider pour passer à l'appairage.

## Détails techniques

### Méthodes de sélection

| Méthode | Geste | Description |
|---------|-------|-------------|
| Sélection simple | Clic | Sélectionne un seul point |
| Multi-sélection | Ctrl + Clic | Ajoute ou retire des points un par un |
| Sélection par plage | Shift + Clic | Sélectionne tous les points entre le premier et le dernier clic |
| Lasso | Clic + glisser | Dessine une zone de sélection rectangulaire |

### Marqueurs détectés par défaut

Les marqueurs suivants sont recherchés dans les noms de points (insensible à la casse) :

| Marqueur | Origine |
|----------|---------|
| stkd | stakeout / staked |
| stked | staked |
| imp | implanté |

### Marqueurs personnalisés

Il est possible de définir des préfixes ou suffixes personnalisés pour correspondre aux conventions de nommage utilisées (ex : `_MES` pour "mesure", `R_` pour "relevé").

### Algorithme de segmentation

L'algorithme découpe le nom du point en segments (séparateurs : underscore `_`, tiret `-`, espace). Si l'un des segments correspond à un marqueur connu, le point est classé comme implanté.

Exemple : `BORNE_A_STKD` → segments [`BORNE`, `A`, `STKD`] → `STKD` est un marqueur → point classé implanté.

### Tri, filtrage et recherche

Les points peuvent être triés par colonne (Nom, X, Y, Z), filtrés par coordonnées (égal à, supérieur à, inférieur à, entre deux valeurs) et recherchés par nom via la barre de recherche.

### Historique d'annulation

L'assignateur conserve un historique de 10 niveaux. Ctrl + Z pour annuler, Ctrl + Y pour rétablir. L'historique couvre toutes les opérations : glisser-déposer, détection automatique, suppressions.
