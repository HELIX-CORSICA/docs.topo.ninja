L'import et l'export CSV vous permettent d'échanger facilement vos données entre Topo.ninja et vos autres logiciels (Excel, Covadis, Leica Infinity, Trimble Business Center, etc.). Cette fonctionnalité est disponible sur tous les plans.

<Note>
  Disponible sur **tous les plans**.
</Note>

## Comment ça fonctionne

### Import

Topo.ninja analyse votre fichier CSV pour détecter automatiquement :
- Le **délimiteur** utilisé (virgule, point-virgule, tabulation)
- Les **en-têtes de colonnes** pour proposer un mapping intelligent
- Le **format des coordonnées** (décimales, DMS, etc.)

Vous n'avez qu'à déposer votre fichier, vérifier le mapping proposé, et valider.

### Export

Après traitement de vos données, vous pouvez exporter les résultats sous plusieurs formes :
- **CSV complet** : toutes les données du projet
- **CSV converti** : coordonnées transformées dans le système cible
- **CSV fusionné** : données sources et résultats dans un seul fichier

## Utilisation

<Steps>
  <Step title="Préparer votre fichier CSV">
    Assurez-vous que votre fichier comporte une **ligne d'en-tête**. Les formats courants sont :
    ```
    Nom,X,Y,Z
    Nom,Latitude,Longitude,Altitude
    Point,Est,Nord,Elevation
    ```
  </Step>
  <Step title="Importer le fichier">
    Deux méthodes possibles :
    - **Glisser-déposer** le fichier directement sur la zone d'import
    - **Cliquer** sur le bouton d'import et sélectionner votre fichier
  </Step>
  <Step title="Vérifier le mapping des colonnes">
    Topo.ninja détecte automatiquement vos colonnes grâce aux en-têtes. Vérifiez que chaque colonne est correctement associée (Nom, X, Y, Z) et ajustez si nécessaire.
  </Step>
  <Step title="Valider l'import">
    Confirmez l'import. Vos points apparaissent dans l'outil en cours d'utilisation (éditeur en masse, assignateur de rôles, etc.).
  </Step>
</Steps>

## Formats de coordonnées supportés

<AccordionGroup>
  <Accordion title="Coordonnées décimales">
    Le format le plus courant. Les coordonnées sont exprimées en nombres décimaux.
    ```
    BORNE_01,654321.123,6789012.456,125.78
    ```
  </Accordion>
  <Accordion title="Degrés Minutes Secondes (DMS)">
    Topo.ninja reconnaît les chaînes DMS et les convertit automatiquement.
    ```
    POINT_A,48°51'24.5"N,2°21'07.0"E,35.2
    ```
  </Accordion>
  <Accordion title="Formats avec séparateurs variés">
    Le délimiteur est détecté automatiquement :
    - **Virgule** : `POINT,1234.56,5678.90,100.0`
    - **Point-virgule** : `POINT;1234,56;5678,90;100,0`
    - **Tabulation** : colonnes séparées par des tabulations
  </Accordion>
</AccordionGroup>

## Options d'export

| Type d'export | Contenu | Cas d'usage |
|---------------|---------|-------------|
| **CSV complet** | Toutes les colonnes du projet | Sauvegarde, archivage |
| **CSV converti** | Coordonnées dans le système cible | Import dans un autre logiciel |
| **CSV fusionné** | Données sources + résultats | Rapport complet, vérification |

## Problèmes courants

<Warning>
  **Séparateur décimal** : en France, le séparateur décimal est souvent la virgule (1234,56) tandis que le séparateur de colonnes est le point-virgule. Si vos coordonnées semblent incorrectes après import, vérifiez que Topo.ninja a bien détecté le bon délimiteur.
</Warning>

<AccordionGroup>
  <Accordion title="Mes colonnes ne sont pas détectées">
    Vérifiez que votre fichier contient bien une **première ligne d'en-tête**. Les en-têtes doivent être descriptifs : `Nom`, `X`, `Y`, `Z`, `Est`, `Nord`, `Latitude`, `Longitude`, etc. Un fichier sans en-tête ne pourra pas être mappé automatiquement.
  </Accordion>
  <Accordion title="Les coordonnées sont inversées">
    Certains logiciels exportent les coordonnées en ordre (Y, X) au lieu de (X, Y). Utilisez le mapping de colonnes pour réassigner manuellement les champs.
  </Accordion>
  <Accordion title="Caractères spéciaux dans les noms">
    Les accents et caractères spéciaux dans les noms de points sont supportés. Si vous rencontrez des problèmes d'affichage, vérifiez que votre fichier est encodé en **UTF-8**.
  </Accordion>
</AccordionGroup>

## Astuces

<Tip>
  **Depuis Excel** : pour exporter en CSV depuis Excel, utilisez "Enregistrer sous" et choisissez "CSV UTF-8 (délimité par des virgules)". Cela garantit une compatibilité optimale avec Topo.ninja.
</Tip>

<Tip>
  **Format minimal** : le format le plus simple accepté est `Nom,X,Y,Z`. Si vous n'avez pas d'altitude, la colonne Z est optionnelle.
</Tip>

<Tip>
  **Gros fichiers** : l'import fonctionne directement dans votre navigateur. Même avec plusieurs milliers de points, le traitement est quasi instantané.
</Tip>
